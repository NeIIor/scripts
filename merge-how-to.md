# Руководство по слиянию веток в Git

## 1. Получение ветки с сервера

### Получение всех обновлений с удаленного репозитория
```bash
git fetch origin

### Получение конкретной ветки с сервераgit checkout -b <имя-ветки> origin/<имя-ветки>
Эта команда:
1. Создает локальную ветку с именем <имя-ветки>
2. Устанавливает связь с удаленной веткой
3. Переключает вас на эту ветку

### Альтернативный способ (если ветка уже существует локально)git checkout <имя-ветки>
git pull origin <имя-ветки>

## 2. Слияние веток

### Основной способ слияния (merge)# Переключитесь на ветку, В КОТОРУЮ хотите влить изменения
git checkout main

# Выполните слияние
git merge <имя-ветки-которую-вливаем>

### Разрешение конфликтов
Если возникли конфликты:
1. Отредактируйте файлы с конфликтами (Git пометит их)
2. Добавьте исправленные файлы:git add <файл-с-конфликтом>
3. Завершите слияние:git commit

### Слияние с перемоткой (rebase)
Альтернативный способ (для линейной истории):git checkout <ваша-ветка>
git rebase main

## 3. Действия после слияния

### Отправка изменений на серверgit push origin <имя-ветки>

### Удаление слитой ветки (если она больше не нужна)# Удаление локальной ветки
git branch -d <имя-ветки>

# Удаление удаленной ветки
git push origin --delete <имя-ветки>

### Обновление основной ветки
Если вы сливали в feature-ветку:git checkout main
git pull origin main
git checkout <ваша-ветка>
git merge main

### Сравнение ветокgit diff main..<ваша-ветка>
